## Frontend build-time environment variables (example)
##
## `frontend/capacitor.config.ts` is evaluated by Node when you run `npx cap sync`.
## Set these env vars before running `npx cap sync` so the native projects get the right IDs.
##
## Why this matters:
## - If GoogleAuth `serverClientId` is wrong, the plugin may return NO `idToken`,
##   and your backend `/auth/google` flow can never work.
##

# Preferred: Google "Web application" OAuth client id (used as serverClientId)
GOOGLE_SERVER_CLIENT_ID=replace-with-google-web-client-id.apps.googleusercontent.com

# Alternate name supported by config:
GOOGLE_WEB_CLIENT_ID=replace-with-google-web-client-id.apps.googleusercontent.com
